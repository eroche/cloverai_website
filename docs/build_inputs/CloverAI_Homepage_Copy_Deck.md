## 1. Homepage objective

The homepage should make a serious visitor understand, quickly and clearly, that:

- Clover.AI is an NLP infrastructure company
- Clover works with LLMs but is not an LLM wrapper
- Clover is built on the CLOVER programming language
- Clover’s first commercial wedge is deterministic anonymization
- Clover extends into broader language-processing capabilities
- the next step is to contact Clover through the form

---

## 2. Core messaging spine

The homepage should follow this progression:

1. broad platform statement  
2. runtime LLM problem framing  
3. first commercial wedge: anonymization  
4. broader platform capability  
5. key differentiation  
6. CLOVER language / technology credibility  
7. restrained founder credibility  
8. final CTA

## 4. Problem / why now section

### Preferred section heading
**LLMs are powerful. Not every text operation should depend on runtime inference.**

### Preferred paragraph
Production AI systems still need a deterministic layer for redaction, routing, extraction, and structure. Clover adds that layer around LLM systems, reducing runtime dependence where language processing can be done faster, more cheaply, and more predictably on CPU.

### Shorter paragraph option
LLMs are powerful, but many production text flows still need a deterministic layer for redaction, routing, extraction, and control. Clover provides that layer.

### Optional bullet points
- Deterministic behavior
- Low runtime cost
- Deployable on CPU
