## 1. Contact page objective

The Contact page should help the right visitor take the next step with as little friction as possible.

It should make clear that Clover wants to hear from people who are evaluating:

- anonymization / redaction
- privacy-safe AI pipelines
- deterministic language processing
- structured text workflows around LLM systems
- broader NLP infrastructure use cases

The page should feel:

- serious
- direct
- technically credible
- low-friction
- custom-solution oriented

It should **not** feel like:

- a generic sales page
- a lead-generation funnel
- a consumer signup form
- an agency “book a free strategy session” page
- a broad customer-support portal

---

## 2. Messaging spine

The Contact page should follow this progression:

1. invitation to describe the use case  
2. reminder of the kinds of workflows Clover can help with  
3. contact form  
4. optional short note about sample data or problem descriptions  
5. simple confirmation state

The page should stay short.

---

## 3. Primary contact page hero

### Preferred heading
**Tell us what text flow you need to control**

### Heading alternatives
**Tell us about your use case**

**Discuss your language-processing workflow**

### Preferred intro paragraph
Evaluating privacy-safe AI pipelines, redaction, extraction, routing, or structured language processing around LLM systems? Send us a short note about your use case.

### Alternative intro paragraph
If you are exploring deterministic anonymization, redaction, or broader language-processing infrastructure, tell us what you are trying to solve.

### Notes
- Use the first heading by default.
- Keep the opening concise.
- Do not add marketing-heavy language.

## 4. Contact page support copy

### Preferred support heading
**Typical reasons to reach out**

### Preferred bullet list
- Evaluating anonymization or redaction for enterprise text flows
- Exploring deterministic language processing around LLM systems
- Discussing extraction, routing, or tokenization use cases

## 6. Form fields

## Required fields
These should appear in this order:

1. **Name**
2. **Work email**
3. **Company**
4. **Use case / problem**
5. **Optional note / message**

### Recommended field labels
- Name
- Work email
- Company
- Use case / problem
- Additional note (optional)

### Placeholder text suggestions

#### Name
`Your name`

#### Work email
`name@company.com`

#### Company
`Company name`

#### Use case / problem
`Briefly describe the text flow, pipeline, or language-processing problem`

#### Additional note (optional)
`Any extra context, timing, sample-data note, or implementation detail`

---

## 7. Helper text and supporting notes

### Preferred helper text under the form
Sample data descriptions are welcome.

### Alternative helper text
If useful, include a short description of the data, workflow, or deployment context.

### Optional short privacy-oriented note
Do not send sensitive raw data unless you are comfortable doing so. A high-level description is usually enough for an initial conversation.

### Important note
Keep helper text minimal.  
Do not add long disclaimers or overly legalistic language unless required.

---

## 8. Primary CTA button

### Preferred CTA
**Send message**

### Acceptable alternatives
- **Contact Clover**
- **Submit inquiry**
- **Request a demo**

### Avoid
- Get started
- Book now
- Talk to sales
- Free consultation
- Start your transformation

### Recommendation
Use **Send message** on the dedicated Contact page.  
Use **Request a demo** when the form is embedded inside a homepage CTA section.

---

## 9. Success / confirmation state

### Preferred success title
**Thank you. Your message has been received.**

### Preferred success body
We will review your note and follow up.

### Alternative shorter success body
Thank you for reaching out.

### Notes
- Keep the success state calm and serious.
- Do not use playful or chatty language.
- Do not promise timing unless that is operationally guaranteed.

---

## 10. Error state suggestions

### Generic error title
**Your message could not be sent.**

### Generic error body
Please try again, or reload the page and resubmit the form.

### Field validation examples
- Please enter your name.
- Please enter a valid work email.
- Please enter your company.
- Please describe your use case or problem.

### Notes
Validation language should be direct and plain.
